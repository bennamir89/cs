check_circle Also called real address

check_circle Performs address translation using page tables

view_modulePaging

view_moduleHow Paging Works

Paging is a memory management technique that divides a process's address space into fixed-size blocks called pages.

check_circle Process divided into equal-sized pages
check_circle Physical memory divided into equal-sized frames
check_circle Page size = Frame size (power of 2)
check_circle Non-contiguous memory allocation

table_chartPage Table

A page table is a data structure that maps logical pages to physical frames.

check_circle One entry per logical page
check_circle Contains frame number and control bits
check_circle Stored in main memory

timerEffective Memory Access Time

Time required to access data in memory:

Without TLB
E.M.A.T = 2 × m
check_circle m = main memory access time
check_circle Two memory accesses: page table + actual data
With TLB (hit ratio = x, access time = c)
E.M.A.T = x(c+m) + (1-x)(c+2m)

categoryHow Segmentation Works

Segmentation is a memory management technique that divides a process into variable-sized logical units called segments.

check_circle Process divided into segments of varying sizes
check_circle Segments represent logical program units (functions, arrays)
check_circle Non-contiguous memory allocation
check_circle Matches user's logical view of program

compareKey Differences

Feature Paging Segmentation
Memory Division Fixed-size blocks (pages) Variable-size blocks (segments)
Address Space One-dimensional Two-dimensional
Fragmentation Internal fragmentation External fragmentation
Table Size Large (one entry per page) Smaller (one entry per segment)
Memory Access Faster with TLB Slower (two memory accesses)
User View System-oriented User-oriented
